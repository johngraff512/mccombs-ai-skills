# Slide Design Principles

Guidelines for creating effective instructional slide decks. These principles apply to both case discussion and lecture slides.

---

## Core Principle: Slides Support the Instructor

Slides are not the presentation — the instructor is. Every slide should serve one of these purposes:

1. **Orient** — tell students what they're about to discuss
2. **Prompt** — give students something to think about or respond to
3. **Display** — show data, frameworks, or visuals that are hard to convey verbally
4. **Record** — capture what the class has discussed (live board-building)

If a slide doesn't do one of these things, it doesn't need to exist.

---

## Slide Count Guidelines

**Less is more.** A 75-minute case discussion typically needs 12-18 slides. A 90-minute lecture might use 15-25.

| Session Type | Duration | Slides |
|--------------|----------|--------|
| Case discussion (short) | 30 min | 6-10 |
| Case discussion (standard) | 75 min | 12-18 |
| Case discussion (extended) | 90 min | 15-22 |
| Lecture (focused) | 50 min | 10-15 |
| Lecture (standard) | 75 min | 15-22 |
| Lecture (extended) | 90 min | 18-25 |

These are guidelines, not rules. An instructor who moves quickly through data-heavy content may need more slides; one who facilitates long discussions may need fewer.

---

## Text Density

**Maximum 3 bullet points per slide.** Each bullet should be a phrase, not a sentence. If you need more text, split across slides.

Bad:
- "Nvidia's competitive advantage stems from its dominance in GPU hardware design, particularly for AI training workloads, where its A100 and H100 chips have become the de facto standard, combined with its CUDA software platform that has accumulated over 4 million developers over 15+ years"

Good:
- Nvidia controls ~80% of AI training chips
- CUDA: 4M+ developers, 15 years of ecosystem lock-in
- FY2025 revenue: $130.5B (up 114% YoY)

## Discussion Question Slides

These are the most important slides in a case discussion deck. Design principles:

- **One question per slide** — large, centered, readable from the back of the room
- **Minimum 28pt font** for the question text
- **Optional subtext** below the question to guide thinking (e.g., "Consider both short-term and long-term implications")
- **No answers on the slide** — the instructor draws answers from students
- **White background preferred** — the question should be the visual focus

## Framework Slides

Pre-built framework templates that the class fills in together during discussion. The slide shows the framework structure; the instructor fills in the content live (or reveals it progressively).

**Built-in framework layouts (strategy-focused):**

| Framework | Layout |
|-----------|--------|
| Five Forces | Center box (rivalry) + 4 surrounding forces |
| VRIO | 4-row table (Valuable, Rare, Inimitable, Organized) |
| Value Chain | Horizontal chain with primary + support activities |
| BCG Matrix | 2x2 grid (Stars, Question Marks, Cash Cows, Dogs) |
| SWOT | 2x2 grid (Strengths, Weaknesses, Opportunities, Threats) |
| Platform Economics | Multi-sided platform diagram |

**Other disciplines:** For frameworks not in the built-in set — such as DCF waterfalls (finance), positioning maps (marketing), process flow diagrams (operations), system architecture diagrams (MIS), or audit risk matrices (accounting) — use `content`, `two_column`, or `data` slides to present the analysis. The `generic` framework type renders labeled key-value rows that work for any discipline.

## Data Slides

When displaying financial or market data:

- **Highlight the insight, not just the numbers** — the slide title should state what the data means, not just what it is
  - Bad title: "Nvidia Revenue 2023-2025"
  - Good title: "Revenue Nearly 5x in Two Years"
- **3-5 data points maximum** per slide
- **Use consistent formatting** (all in $B, all in %, etc.)
- **Always include the source** in small text at the bottom

---

## Visual Hierarchy

Every slide should have a clear reading order:

1. **Title** (what this slide is about)
2. **Main content** (the key information)
3. **Supporting detail** (footnotes, sources)

Achieve this through:
- Size contrast (title larger than body)
- Color contrast (Burnt Orange for titles, Charcoal for body)
- Position (title at top, content in middle, footnotes at bottom)
- Whitespace (don't fill every pixel)

---

## Color Usage

### Background Patterns

| Slide Type | Background | Text Color |
|------------|-----------|------------|
| Title slide | Burnt Orange | White |
| Section divider | Burnt Orange | White |
| Content slide | White | Charcoal/Black |
| Question slide | White | Burnt Orange (question), Charcoal (subtext) |
| Takeaway slide | White | Charcoal |

### Emphasis

- Use Burnt Orange for emphasis, not bold (bold is hard to see at distance)
- Use Steel Gray for secondary information
- Never use red for emphasis — it implies error or danger

## Case Discussion Deck Structure

A typical case discussion deck follows this arc:

### 1. Opening (1-2 slides)
- Title slide
- Case setup: protagonist, situation, decision point

### 2. Analysis (6-10 slides)
- Discussion question → supporting data/framework → discussion question → ...
- Build complexity progressively
- Each question-data pair = one discussion block

### 3. Synthesis (2-4 slides)
- Strategic options comparison
- Vote/debate prompt
- Key trade-offs

### 4. Close (1-2 slides)
- Key takeaways
- Next class prep

---

## Common Mistakes

- **Too many words** — if you're reading the slide, there's too much text
- **Orphan slides** — a slide with no clear purpose in the discussion flow
- **Missing prompts** — data slides without a question to drive discussion
- **Framework overload** — applying 4+ frameworks makes the deck feel academic rather than practical
- **No data** — discussion with zero quantitative grounding feels like opinion-swapping
- **Inconsistent formatting** — mixing font sizes, colors, or alignment across slides
