# Case Refresher

Update an existing business case with current data, recent developments, and new dynamics relevant to the case's discipline. This skill works across all McCombs departments — Strategy, Marketing, Finance, Operations, MIS, and Accounting. See `../shared/discipline-guide.md` for department-specific materiality thresholds and update categories.

## Description

Use this skill when an instructor has an existing case — either one generated by the Case Generator or any published case (HBS, Ivey, Darden, custom) — and wants it refreshed with current information. The skill researches what has changed since the case was written and produces either a full updated case edition (from Case Generator JSON) or a refresh memo (from any case document).

**Trigger phrases:** "refresh this case," "update the case," "what's changed since this case was written," "case refresher," "bring this case up to date," "current data for this case"

## Two Output Modes

### Mode A: Updated Case Edition (Case Generator JSON input)

When the input is a Case Generator JSON file, the skill can produce a **fully updated version** of the case — same structure, same pedagogical design, but with:
- Updated financial data and metrics
- New competitive developments incorporated into the narrative
- Revised exhibits with current numbers
- Updated sources list
- A note on what changed (appended as a version note)

This mode uses the Case Generator's `generate_case_docx.js` script to produce the output.

### Mode B: Refresh Memo (Any case input)

When the input is an uploaded case document (PDF, .docx, or any format), the skill produces a **Case Refresh Memo** — a standalone companion document the instructor can use alongside the original case. The memo covers:
- What has changed since the case's decision point
- Updated financial and competitive data
- New developments that affect the case's central dilemma
- Whether the dilemma is still live, has been resolved, or has evolved
- Suggested discussion adjustments for instructors

This mode uses its own `generate_refresh_memo_docx.js` script.

## Workflow

### Phase 1: Gather Inputs

Determine the input type and collect context:

**Required:**
- The case — either a Case Generator JSON file or an uploaded case document
- When the case was written (or approximate date of the data in the case)

**Helpful:**
- The company/companies covered in the case
- The central dilemma or decision point
- What specific areas the instructor wants refreshed (financials, competitive landscape, regulatory, all)

If the instructor provides a Case Generator JSON, all of this information is already embedded in the file.

### Phase 2: Research Current State

Use **web search** to research what has happened since the case was written. Focus on:

1. **Financial updates** — latest quarterly/annual results, revenue, margins, growth rates, valuation changes
2. **Competitive/market developments** — new entrants, acquisitions, product launches, market share shifts (strategy); credit rating changes, capital markets shifts (finance); customer preference changes, competitive repositioning (marketing); supply chain disruptions, technology changes (operations); platform evolution, security threats (MIS); new standards, enforcement actions (accounting)
3. **Actions by the protagonist** — what has the company/decision-maker actually done since the case's decision point?
4. **Regulatory/policy changes** — new regulations, rulings, or policy shifts that affect the case
5. **Industry trends** — broader market developments that reframe the case's context
6. **Resolution (if any)** — has the dilemma been resolved? Did the protagonist choose one of the paths discussed in the case?

For each finding, note the source and date.

### Phase 3: Assess Impact

Evaluate each development against the case's pedagogical goals:

| Impact Level | Meaning | Action |
|-------------|---------|--------|
| **High** | Changes the central dilemma or makes an option clearly wrong/right | Must include — may require narrative revision |
| **Medium** | Adds important context but doesn't change the core decision | Include as updated data or new information |
| **Low** | Minor update, interesting but not essential | Include only if space permits |

Also assess:
- **Is the dilemma still live?** If the company has made its choice, the case becomes historical — still teachable, but the discussion shifts
- **Has the competitive landscape changed materially?** New competitors, exits, or market shifts
- **Are the case's data points now significantly outdated?** Revenue figures 2+ years old, market share numbers that have shifted meaningfully

### Phase 4: Generate the Output

**For Mode A (Updated Case Edition):**

1. Create an updated version of the Case Generator JSON
2. Update financial data in exhibits
3. Revise narrative sections to incorporate developments
4. Add/update sources
5. Add a version note at the end of the JSON: `"versionNote": "Updated [Month Year] with [summary of changes]"`
6. Generate the .docx using the Case Generator script:
   ```bash
   node /path/to/skills/case-generator/scripts/generate_case_docx.js <updated.json> <output.docx>
   ```

**For Mode B (Refresh Memo):**

Create a JSON object following the schema below, then generate the .docx:

```json
{
  "title": "Case Refresh Memo",
  "meta": "McCombs School of Business  |  [Date]",
  "caseName": "Original case title",
  "caseDate": "When the case was written/published",
  "refreshDate": "When this memo was generated",
  "company": "Company name",
  "protagonist": "Protagonist name and title",
  "summary": "Brief summary of the original case's central dilemma (2-3 sentences)",
  "dilemmaStatus": {
    "status": "Live | Resolved | Evolved",
    "explanation": "How the dilemma has changed or been resolved"
  },
  "updates": [
    {
      "category": "Financial Performance | Competitive Landscape | Strategic Actions | Regulatory | Industry Trends",
      "headline": "One-line summary of the update",
      "detail": "2-3 sentences explaining the development and its significance",
      "impact": "High | Medium | Low",
      "source": "Source name and date",
      "caseImplication": "How this affects the case discussion"
    }
  ],
  "revisedExhibits": [
    {
      "title": "Exhibit title (e.g., Updated Revenue Data)",
      "headers": ["Column 1", "Column 2"],
      "rows": [["Data", "Data"]]
    }
  ],
  "teachingImplications": [
    "How the instructor should adjust their approach given these updates",
    "New discussion questions that arise from recent developments",
    "Which case arguments are stronger/weaker now"
  ],
  "sources": [
    {
      "number": 1,
      "citation": "Full source citation"
    }
  ]
}
```

Then run:
```bash
node /path/to/skills/case-refresher/scripts/generate_refresh_memo_docx.js <input.json> <output.docx>
```

### Phase 5: Review and Iterate

Present the output. The instructor may want to:
- Focus on specific areas (e.g., "just update the financials")
- Add or remove updates
- Adjust the impact assessment
- For Mode A: revise specific narrative sections
- For Mode B: add teaching implications

## Quality Standards

- [ ] Every data point includes a source and date
- [ ] Financial data is the most recent available (not just any newer data)
- [ ] The impact assessment is honest — "Low" impact items shouldn't be inflated
- [ ] The dilemma status assessment is nuanced, not binary
- [ ] Teaching implications are practical, not generic
- [ ] The memo can stand alone — an instructor who hasn't read the original case recently should be able to follow it
